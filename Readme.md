## Перечень используемых инструментов с обоснованием выбора

    1. GitHub -  для хранения кода и автотестов
    1. JUnit5 - платформа для написания автотество и их запуска, наиболее широко используемая среда тестирования для приложений Java
    1. Java 11  - широко используемый язык написания автотеств
    1. IDEA - удобная и самая популярная в мире среда разработки. Обеспечивает компиляцию, запуск на исполнение, отображение ошибок, автоподсказки, подсветку синтаксиса, генерацию кода и т.д.
    1. Gradle - универсальная система управления проектами, автоматизации сборки и управления зависимостями. Эта система сама скачивает необходимые для написания тестов библиотеки и зависимости этих библиотек
    1. Selenide -фреймворк для автоматизированного тестирования вебприложений, имеющий такие преимущества как, изящный API, поддержка Ajax для стабильных тестов, мощные селекторы, простая конфигурация.
    1. Docker - система контейнеризации, которая поможет работать с базой данных
    1. Docker compose - удобрый инструмент, позволяющий запускать мультиконтейнерные приложения.
    1. Allure -  фреймворк, предназначенный для создания визуально наглядной картины выполнения тестов.


## Перечень необходимых разрешений/данных/доступов

    1. Разрешение на тестирование сайта;
    1. Данные для тестирования - требования к имени и номеру телефона для записи на курс;
    1. Доступ к базе данных со списком записывающихся на курс для создания зарегистрированного тестового пользователя и записи нового тестового пользователя или схема таблицы для разворачивания бд на компьютере тестировщика;
    1. Доступ к базе данных со списком зарегистрированных пользователей для создания зарегистрированного тестового пользователя или схема таблиц для разворачивания бд на компьютере тестировщика.


## Перечень и описание возможных рисков при автоматизации

    1. Утечка информации из базы данных или изменение данных в базе в процессе тестирования;
    1. При использовании схемы бд, возможно,что настоящая бд может повести себя подругому;
    1. Автоматизация может занять много времени; 
    1. Увеличивается стоимость тестирования;
    1. Если разработчики изменят код (например подругому назовут селекторы, задействованные в тестах), то тесты перестанут работать;
    1. Необходимо время на поддержку автотестов;


## Перечень необходимых специалистов для автоматизации

    1. QA, занимающийся разработкой тестов:
        * Разработка и отладка тестов
        * Поддержка, обновление
        * Исправление ошибок

    1. QA, занимающийся тест-дизайном и управлением:
        * Планирование
        * Отбор тестов
        * Проектирование тестов
        * Проектирование тестовых данных

    1. Тестировщик, занимающийся непосредственно тестированием:
        * Локализация ошибок
        * Заведение ошибок
        * Подготовка тестовых данных
